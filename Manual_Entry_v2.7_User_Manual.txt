3. MEASUREMENT SETUP
============================================================

UNITS CONFIGURATION:

Configure measurement units for each category before starting data collection:

Height Measurements: Inches (default) or Centimeters
Jump/Distance Measurements: Inches (default) or Centimeters
Weight: Pounds (fixed)
Hand Measurements: Inches (fixed)

ADJUSTMENT SYSTEM:

Adjustments are systematic corrections applied to all measurements based on equipment calibration, environmental factors, or standardization requirements.

HOW ADJUSTMENTS WORK:
1. Gender-Specific: Different adjustments for Male (M) and Female (F) participants
2. Additive: Adjustments are added to raw measurements
3. Automatic: Applied during data collection and export
4. Measurement-Specific: Each measurement type has its own adjustment value

AVAILABLE ADJUSTMENTS:
‚Ä¢ Height with Shoes - Standing height wearing shoes
‚Ä¢ Height without Shoes - Standing height barefoot/socks
‚Ä¢ Reach - Standing reach measurement
‚Ä¢ Wingspan - Arm span measurement
‚Ä¢ Weight - Body weight
‚Ä¢ Hand Length - Wrist to fingertip
‚Ä¢ Hand Width - Palm width
‚Ä¢ Vertical Jump - Maximum vertical leap
‚Ä¢ Approach Jump - Running vertical leap
‚Ä¢ Broad Jump - Standing horizontal jump

SETTING ADJUSTMENTS:
1. Enter values in the Measurement Adjustments table
2. Use positive numbers to add to measurements
3. Use negative numbers to subtract from measurements
4. Default value is 0 (no adjustment)
5. Adjustments automatically save when changed

EXAMPLE: If your measuring tape consistently reads 0.5 inches low due to equipment calibration, enter +0.5 in the appropriate adjustment field to correct all measurements automatically.

4. DATA COLLECTION
============================================================

MEASUREMENT PROCESS:

1. Select Participant
   - Click on a participant card from the roster grid
   - Card will expand to show measurement interface

2. Enter Measurements
   - Each measurement requires two identical entries for validation
   - Enter the first value and press Enter or Tab
   - Enter the second value to confirm
   - Values must match exactly to save

3. Save Process
   - Measurements auto-save when both entries match
   - Green highlighting indicates successful validation
   - Red highlighting indicates entry mismatch

MEASUREMENT DEFINITIONS:

Height with Shoes: Standing height wearing shoes
- Technique: Participant stands straight against wall, shoes on

Height without Shoes: Standing height barefoot or in socks
- Technique: Participant stands straight against wall, no shoes

Reach: Standing reach (fingertip height)
- Technique: Participant reaches up with dominant hand

Wingspan: Arm span (fingertip to fingertip)
- Technique: Arms extended horizontally, measure tip to tip

Weight: Body weight in pounds
- Technique: Standard scale measurement

Hand Length: Length from wrist to fingertip
- Technique: Measure from wrist crease to middle fingertip

Hand Width: Width across palm
- Technique: Measure across widest part of palm

Vertical Jump: Maximum vertical leap from standing
- Technique: Standing jump, measure highest point reached

Approach Jump: Maximum vertical leap with running start
- Technique: Running approach jump, measure highest point

Broad Jump: Standing horizontal jump distance
- Technique: Standing position, jump forward, measure distance

CALCULATION DETAILS:

Adjusted Value Calculation:
Final Value = Raw Measurement + Gender-Specific Adjustment

Unit Conversions:
‚Ä¢ All calculations performed in inches internally
‚Ä¢ Display units converted based on your selection
‚Ä¢ Height displays include feet/inches format (e.g., "6' 2"")

VALIDATION RULES:
‚Ä¢ Both entries must be identical to the decimal place
‚Ä¢ Values must be positive numbers
‚Ä¢ Empty entries are allowed (saves as blank)
‚Ä¢ System prevents impossible values (e.g., negative measurements)

11. TROUBLESHOOTING
============================================================

COMMON ISSUES:

Problem: App Won't Load Roster
Causes: CSV format incorrect, special characters, file extension
Solutions: Verify CSV format, use quotes for special characters, ensure .csv extension

Problem: Measurements Won't Save
Causes: Values don't match, negative numbers, no participant selected
Solutions: Ensure both entries identical, use positive numbers, select participant first

Problem: Export Not Working
Causes: Storage space, permissions, browser issues
Solutions: Check storage space, verify Downloads folder access, try different browser

Problem: App Runs Slowly
Causes: Large roster, memory issues, other apps
Solutions: Close other apps, restart browser, reduce roster size if extreme

DATA RECOVERY:
‚Ä¢ Auto-Backup: App maintains automatic backups
‚Ä¢ Manual Backup: Export data regularly as protection
‚Ä¢ State Recovery: Restart app to restore last session
‚Ä¢ Emergency Export: Use View Saved Files if export button fails

SUPPORT INFORMATION:
‚Ä¢ Error Logging: Check browser console for detailed error messages
‚Ä¢ Version Info: App displays current version (v2.7) in header
‚Ä¢ Device Info: Automatically included in export files for support
‚Ä¢ Data Privacy: All troubleshooting can be done without sharing personal data

12. BEST PRACTICES
============================================================

BEFORE STARTING:
1. Test Setup: Try with small sample roster first
2. Verify Equipment: Ensure measurement tools are calibrated
3. Check Settings: Configure units and adjustments before data collection
4. Backup Plan: Know how to export data in case of device issues

DURING DATA COLLECTION:
1. Regular Exports: Export data every 50-100 measurements
2. Double-Check: Review measurements before moving to next participant
3. Consistent Process: Use same measurement technique for all participants
4. Break Periods: Take breaks to maintain measurement accuracy

AFTER COMPLETION:
1. Final Export: Ensure all data is exported
2. Verify Export: Open CSV file to confirm data integrity
3. Backup Storage: Save exported files to multiple locations
4. Reset App: Clear data if device will be reused for different event
